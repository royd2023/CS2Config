## Component Inventory

### CrosshairPreview (new — Phase 2)
- SVG element inside a div with background `#1a1a1a`
- Reads from `useCrosshairStore` directly
- Scale rule: 1 CS2 unit = 4px at 1080p reference. `cl_crosshairsize 4` → 16px arm length each side.
- Gap: `cl_crosshairgap` maps 1:1 to px (negative values pull arms inward toward center)
- Thickness: `cl_crosshairthickness` maps 1:1 to px (minimum 1px rendered)
- Outline: rendered as a slightly larger duplicate arm/dot underneath, color black, offset by `outline.thickness` px
- Center dot: rendered as a square at center, size derived from `cl_crosshairsize` scaled
- T-style: top arm is omitted from render
- Style 0/1 (Default/Default Static): render same as Classic Static for preview purposes (share code only supports styles 2–5)
- Color: `rgba(r, g, b, a/255)` CSS value — alpha maps from 0–255 to 0–1

### Slider (Phase 1 primitive — styled in Phase 2)
- Apply Tailwind styles to the existing unstyled `Slider` component via className prop or wrapper
- Range input track: `#3a3a3a` background, 4px height, border-radius 2px
- Range input thumb: 16px × 16px, `#e5e5e5` fill, border-radius 50%, focus ring in accent color
- Companion number input: same border/bg as section card (`#3a3a3a` border, `#1a1a1a` bg), 4px border-radius, right-align text

### SegmentedButtonGroup (new — Phase 2)
- 4 buttons in a horizontal flex row, full-width of section card
- Borders: 1px `#3a3a3a`, no gap between buttons (shared border collapse)
- Border-radius: 6px on first and last button only
- Inactive: bg `#262626`, text `#e5e5e5`, weight 600, 14px
- Active: bg `#f97316`, text `#1a1a1a`, weight 600, 14px
- Hover (inactive only): bg `#2e2e2e`
- Height: 40px
- Switching style resets all params (locked — CONTEXT.md)
- Controls not applicable to selected style are hidden with `display: none` (not `opacity: 0` or `disabled`) — locked — CONTEXT.md

### ColorSwatch (new — Phase 2)
- 24px × 24px block, border-radius 4px
- Checkerboard pattern behind it (CSS `background-image` with two linear-gradient repeating tiles at 6px) to show alpha transparency
- Color layer on top as a pseudo-element or inner div with `rgba(r, g, b, a/255)`
- 1px border `#3a3a3a`
- Not interactive — display only

### ColorSection (new — Phase 2)
- Top row: ColorSwatch (24px) + 8px gap + current RGBA readout label (`R: 0  G: 255  B: 0  A: 200`) in 13px mono
- Rows: R Slider (0–255), G Slider (0–255), B Slider (0–255), Alpha Slider (0–255)
- No color presets (locked — CONTEXT.md)
- No native `<input type="color">` picker (locked — CONTEXT.md note: ColorPicker primitive is hex-only; Phase 2 color section uses R/G/B/A sliders only)

### ToggleSwitch (new — Phase 2)
- Native `<input type="checkbox">` visually replaced with a CSS pill toggle
- Track: 36px × 20px, border-radius 10px
- Thumb: 16px circle, white, with 2px inset from track edges
- Checked state: track bg `#f97316`, thumb slides to right
- Unchecked state: track bg `#3a3a3a`, thumb at left
- Transition: 150ms ease on track bg and thumb transform
- Focus ring: 2px `#f97316` outline, 2px offset

### ImportCodeBar (new — Phase 2)
- Always visible at bottom of controls (locked — CONTEXT.md)
- Full-width container, bg `#262626`, border 1px `#3a3a3a`, border-radius 8px, padding 16px
- Label: "CS2 Import Code" — 12px / semibold / `#737373` / uppercase
- Code display: `<code>` element, 13px mono, `#e5e5e5`, wraps on overflow
- CopyButton: right-aligned, same row as code on desktop, below on narrow viewports
- On copy success: button text changes to "Copied!" in `#22c55e` for 2 s (Phase 1 CopyButton behavior — no change needed)

### CopyButton (Phase 1 primitive — styled in Phase 2)
- Normal state: bg `#262626`, border 1px `#3a3a3a`, text `#e5e5e5`, border-radius 6px, px 12px py 8px
- Hover: bg tint `#f97316` at 15% opacity (`rgba(249,115,22,0.15)`), border-color `#f97316`
- Success state: text `#22c55e`, border-color `#22c55e`
- Label text: "Copy Code" (normal) / "Copied!" (success)

## Interaction Contract

### Real-time preview
- Every store state change triggers an immediate SVG re-render (React reactive — no debounce)
- No loading states, spinners, or skeleton screens — all computation is synchronous

### Style switching
- Clicking a style button immediately sets style + resets all crosshair params to CS2 defaults for that style
- Reset is silent — no confirmation dialog (locked — CONTEXT.md)
- Hidden controls disappear with no animation (display: none, instant)

### Slider + number input sync
- Dragging slider updates number input value in real time
- Editing number input updates slider position on blur (or Enter key)
- Invalid / out-of-range entry in number input: clamp to min/max on blur, no error message shown

### Toggle switch
- Click anywhere on the row (label + toggle) toggles the boolean
- No secondary confirmation

### Copy button
- Click writes share code string to clipboard via `navigator.clipboard.writeText`
- Success flash: 2 s "Copied!" state (Phase 1 implementation — no change)
- No failure state shown to user (clipboard API failure is silent console.error per Phase 1)
